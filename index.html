01. Discover

业务背景与挑战

集互通是一款面向商家的综合电商管理平台，涵盖商品、客户、数据、客服等核心模块。随着业务体量的增长，旧版平台暴露出明显的体验瓶颈：操作流程冗长、信息架构模糊、数据价值未被充分挖掘，直接影响了商家的经营效率。

商品上架慢，容错率低

商家反馈商品发布流程复杂，信息填写繁琐，规格设置不够灵活，难以满足多样化商品的管理需求。

客资分散，跟进断层

客户信息缺乏统一视图，客服接待时无法快速调取客户画像与历史订单，导致沟通成本高昂。

数据堆砌，决策困难

报表展示不够直观，关键指标被淹没在海量数据中，商家难以快速获取洞察以指导后续运营。

权限割裂，协同低效

面对不同规模的商家团队，原有的权限管理颗粒度粗，分配逻辑不清晰，增加管理成本。

02. Define

洞见用户：寻找体验的破局点

针对不同发展阶段的商家，我们需要提供差异化但同样高效的体验支撑。

01

小型商家店主

极致效率 / 低学习成本

通常一人身兼多职（上架、客服、发货）。他们对操作便捷性要求极高，没有时间学习复杂系统，关注平台能否快速帮其促成交易。

02

中型商家运营

精细化营销 / 数据驱动

拥有明确的团队分工。需要精准的数据分析来制定营销策略，对客户分层管理、复盘报表有高频且深度的使用诉求。

03

大型商家主管

全局掌控 / 安全协同

不直接参与一线操作，更注重平台的稳定性、安全性、组织权限分配以及宏观业务数据的整合分析能力。

04. Design System

构建理性与温度并存的规范

B端设计不仅需要严谨的逻辑，也需要在高频操作中注入品牌温度。我们以“暖橙”为核心，构建了一套克制而清晰的视觉语言。

色彩系统 Color System

Brand Primary

#F28E2B

Text Main

#1F2937

Bg Base

#F9FAFB

摒弃了传统B端冷峻的蓝色系，采用暖橙色作为主色调，传递“服务、热情、活力”的平台属性。在辅色和背景色上保持极致的克制（大面积白与灰），确保数据内容的主体地位不受干扰。

栅格与布局 Grid & Layout

全面采用响应式 24 栅格系统（示意缩略为12），水槽设定为 16px/24px。卡片采用 8px 圆角（适中的亲和力），内边距遵循 4 的倍数规律（16/24/32px），保证信息的呼吸感与阅读的流畅性。

05. Core Experience 1

全局驾驶舱：从“看数据”到“用数据”

原版首页信息平铺直叙。新版设计引入了“层级渐进”与“场景聚焦”理念。首屏最高优先级展示核心指标（北极星指标）与高频待办，利用 F 型视觉动线引导商家快速掌握当日经营盘面。

F型动线布局
顶部数据概览 → 中部待办事项 → 底部图表分析，符合人眼自然扫描习惯。
卡片式封装
高内聚低耦合，不同业务模块通过卡片隔离，减少视觉噪音。

数据降噪与层级

提取“今日总览”、“待办事项”置于第一屏绝对核心区。通过大小对比、色彩对比，让商家第一眼就能抓住“今天赚了多少”、“有什么急需处理”。

快捷入口的场景化

摒弃了冗长的左侧边栏寻找过程，在首页中部提供高频操作的“便捷入口”（如商品发布、订单查询），并支持用户自定义，真正实现“千人千面”的效率提速。

右侧赋能区

右侧区域不放置主业务流，而是作为“赋能区”：店铺基础信息、系统公告以及“专属客服助手”，提供陪伴式服务体验。

报表分析：让数据趋势“说话”

在深度数据分析（如“我的报表”）场景中，设计的核心是**可视化的直观度**。我们避免了干瘪的表格罗列，大量运用微交互图表。

  • 1

    业务监控仪表盘： 顶部采用渐变暖色卡片包围，利用双环形进度条直观展示“业绩完成度”与“时间进度”的对比，产生视觉推力。

  • 2

    时段访问平滑曲线： 放弃生硬的折线，采用贝塞尔平滑曲线展示流量趋势，配合悬浮态的数据点透出，既美观又专业。图例交互支持动态显示/隐藏特定维度。

  • 3

    关键指标异动提示： 转化率、退款件数等核心指标附带“较昨日”的红绿升降标识，利用颜色阈值快速拉响业务警报。

06. Core Experience 2

商品发布：化繁为简的向导式体验

传统的商品上架表单往往是一个“无底洞”般的长页面，极易引起用户的认知疲劳和操作中止。我们采用了“任务拆解”的逻辑重塑此流程。

向导式进度条 (Stepper)，清晰预期

认知减负：向导分步设计

将多达数十项的表单字段，根据逻辑关联拆解为 3 个清晰的步骤：“选择分类 → 基本信息 → 发布成功”。顶部的分步指示器（Stepper）始终可见，给予用户明确的位置感和任务预期，有效降低了由于页面过长导致的流失率。

交互优化：多媒体与长表单处理

商品主图上传区域采用大面积的拖拽热区，支持批量操作；导购标题等字段提供“标签库”快速插入功能，变“打字输入”为“点击选择”。必填项红星标记清晰，底部固定操作栏（保存草稿/下一步）确保随时可触达，防止数据丢失。

07. Core Experience 3

客户管理：多维筛选与统一视图

B端表格承载着最高频的数据处理任务。对于“客户概况”页，设计的关键在于“高效检索”与“清晰展示”的平衡。

表格设计的巧思

  • 复合筛选区： 顶部 Tab 预置了“普通/优质/潜在”等常用分层，降低高频筛选的操作步长。下拉框与搜索框组合，形成漏斗式条件检索，点击“查询”触发，避免频繁刷新干扰。
  • 信息降噪与对比： 表格去除了繁重的纵向分割线，利用斑马纹或简单的横线引导视线。状态字段（如：进行中、已完成、逾期）采用明确的色彩标识符（Dot），一目了然。
  • 批量操作收敛： 列表上方的操作栏（导出、批量编辑等）在未勾选数据时保持相对弱化，勾选后可通过悬浮或高亮形式引导，保持界面清爽。

08. Core Experience 4

工作台视图：赋能一线的智能客服

客服接待是一个极度追求“时间效率”的场景。我们调研发现，客服经常需要在聊天窗口、订单系统、客户CRM之间来回切换，严重拖慢了响应速度（ART）。

设计解法：三栏式沉浸式工作台

将会话列表、沟通视窗、客户画像面板整合在同一层级，实现“所见即所需”。

  • 左侧（排队与列表）： 紧凑的会话列表，实时展示倒计时/未读数，利用视觉紧迫感督促响应。
  • 中栏（沟通场域）： 极简的对话流设计，提供快捷回复、订单卡片发送等富文本工具，减少键盘敲击次数。
  • 右侧（全息画像）： 此次优化的核心亮点。 自动关联当前用户的标签（如高价值、VIP）、基础备注、以及最近订单明细及物流状态。客服无需询问单号，即可主动掌握服务上下文，提供“懂你”的服务体验。

09. Core Experience 5

复杂逻辑的优雅表达：账号权限

中大型商家对账号权限的管理颗粒度要求极高。面对庞大如树状图的权限节点，如果设计处理不当，极易造成页面混乱和配置错误。

管理范围与功能解耦

将权限分为宏观的“管理范围”（店铺、仓库、品牌维度）与微观的“功能权限”（具体能点哪些菜单）两大区域。顶部用表单定基调，下方用 Tab 切换不同类型的细分权限，逻辑层次分明。

模块化复选逻辑

在功能权限分配区，采用“模块分组 + 复选框”的形式。例如“基础档案”下一组，“商品策略”下一组。背景色的交替使用（浅灰斑马纹区块）有效隔绝了不同模块组，防止视觉串行。右上角提供“全选/反选”快捷操作，极大提升了主管的配置效率。
